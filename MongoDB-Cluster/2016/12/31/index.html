NoSQL概述

随着互联网的高速发展,传统的面向关系型数据库渐渐无法满足高速实时 PB级 非结构化数据的存储.很快Not Only SQL数据库开始得到普遍的应用,如:面向文档的MongoDB,面向列的HBase/Cassandra,面向key-value的Redis,以及基于图像的Neo4j等.但是NoSQL旨在强调Key-Value和文档数据库的优点,并非单纯的反对关系型数据库.

MongoDB

MongoDB是一种基于文档的开源数据库,具有高可用,可扩展和高性能的特点.

Document Database

MongoDB中一条记录对应一个文档,可以理解 成key-value键值对的数据结构.文档与JSON对象非常相似,一个文档中可以包括其它文档,数组.这里将MongoDB与SQL中的概念进行罗列与类比.























SQL术语/概念MongoDB术语/概念解释/说明
DATABASEdatabase数据库
TABLEcollection数据库表/集合
ROWdocument数据记录行/文档
COLUMN field 数据字段/域
INDEX index 索引
TABLE JOINS 表连接,MongoDB不支持
PRIMARY KEY primary key 主键,MongoDB自动将_id字段设置为主键

主要优点

High Performance 高性能

MongoDB高性能体现在数据持久化上,其对嵌入式数据模型的支持降低了数据库系统的I/O操作,而索引库使得查询其在数据查询上表现尤为出色.

Rich Query Language 丰富的查询语言

MongoDB不但支持简单的CRUD操作,还支持用户进行数据聚合,文本搜索以及基于地理位置的查询.

High Availability 高可用

MongoDB的数据高可用系统叫:replica set.它是一个由一组存有相同数据的镜像MongoDB服务组成,这一组服务器集群可以在主节点宕机时,自动完成选举迅速对外恢复访问接口;同时保证了单点访问的尴尬;集群中数据一致性通过主节点的log文件实现,所有从节点抓取日志,并将日志中的操作加载到内存中,从而保证数据一致性.

Horizontal Scalability 水平扩展

所谓水平扩展是相对垂直扩展而言的.通常,当系统性能不足时,如硬盘空间不足或内存吃紧时,很容易想到给服务器升级硬件,但但往往硬件的升级会带来相对较高的成本,而且是无止境的.试想,GOOGLE的每一台服务器都是相同配置的硬件,那么GOOGLE就太不会过日子了.

除了升级硬件,往往会选择多加几台配置相对较低的服务器,这就是水平扩展.MongoDB的replica sets对水平扩展有很好的支持,只需要在一台服务器上装上MongoDB,让它与其它replica sets在可达的网段中,启动这台server的mongod服务即可.
